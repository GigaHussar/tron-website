Local LLMs That Can Search the Web

May 2026

#ai#automation#web

There's a lot of noise about AI agents right now. I came across Open Claw — an open-source agentic framework — and thought it looked genuinely impressive. But I didn't have a clear use case for it at the moment. What I did want to try was something more specific: a local LLM generating research reports by actually searching the web. No cloud API, no data leaving the machine.

I already knew I'd use Ollama or LM Studio to serve the model. The question was what to put in front of it.

My first try was Open WebUI. By default it nudges you toward getting an API key from one of the supported search providers. I just wanted to test the concept quickly before committing to anything long-term, so I picked DuckDuckGo — no key required. It didn't work. Rate limited immediately. I dug into the settings and tried adding delays between requests, but the option didn't seem to do anything. Then I found a Python tool on GitHub specifically built to wrap DuckDuckGo requests with proper delays — and it could be imported as a plugin. That worked. But at some point I stepped back and thought: this is getting more complicated than it needs to be.

So I looked for something else.

The second tool I tried was Local Deep Research — a fairly new project. Out of the box, without any API keys or a self-hosted search engine, it's limited to a handful of websites. But that was enough to confirm the basic loop worked and felt usable. Good enough to commit to the proper setup.

I followed the recommended path: SearXNG running locally in Docker. SearXNG is a self-hosted meta search engine — it queries multiple search providers simultaneously and returns the results to whatever tool is asking, without any single provider tracking your queries. The setup instructions were clear and I didn't run into any problems. Once it was running, I just pointed Local Deep Research at the local SearXNG address and everything clicked into place.

For the model I'm using LM Studio with Google Gemma 4 E4B. It handles the research prompts well enough, though I plan to experiment with other models down the line.

The final touch was a .bat script that launches all three pieces in sequence — SearXNG in Docker, LM Studio with the model loaded, and Local Deep Research — so the whole stack starts with a single double-click. It's not fancy, but it means I actually use the thing instead of having to remember the startup order every time.
